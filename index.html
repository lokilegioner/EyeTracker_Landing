Medical application for the eyes

The application main function is tracking the speed and coordinates of eye pupils via the mobile device camera

This is important to know

The EyeTracker in the native application built with the OpenCV library integration, which helps with programming functions mainly aimed at real-time computer vision.

The application is making approximately 1 photo of a patient's face with the 0.5 seconds frequency.

The photos of the patient's face can also be uploaded and processed.

All the computing is processed on the native app side, with no back-end.

Photo processing and process costs

Before starting the process of finding the coordinates of pupils, the taken or uploaded photos should be reduced to the minimum resolution in order to increase the speed of processing them by the OpenCV library

Data output and charts

As a result, the application is building the graphics with the eye position which allows a doctor to make a conclusion with ease. EyeTracker stores the doctor registration data and a list of his patients with their medical data. The main purpose of EyeTracker is help saving the time of medical observation and identification of eye problems.
